# /客訴
> 🏷️ **Skills UI 英文名稱（方法一手動建立時填入）**：`complaint`
> 📌 **觸發詞（方法二動態載入後使用）**：輸入 `/客訴`

## 指令
讀取這份客訴內容（Email、LINE 截圖或文字檔），幫我產出一封對外回覆信和一份對內改善追蹤表。

## 執行規則

### 對外回覆信（Word）
- 語氣：誠懇、專業、積極解決問題，避免推諉
- 結構：
  1. 感謝客戶反映 + 致歉
  2. 說明初步判斷的原因（根據客訴內容合理推測，標註「初步研判」）
  3. 具體改善措施與時程承諾
  4. 後續聯繫窗口資訊（留空讓使用者填入）
- 檔名格式：`客訴回覆_客戶名稱_YYYY-MM-DD.docx`

### 對內改善追蹤表（Excel）
- 欄位：編號、客訴日期、客戶名稱、產品型號、問題描述、根因分析（待填）、改善對策（待填）、負責人（待填）、預計完成日（待填）、狀態
- 狀態預設為「🔴 待處理」
- 若客訴中提到多個問題，每個問題獨立一列
- 檔名格式：`客訴追蹤_客戶名稱_YYYY-MM-DD.xlsx`

### 共通規則
- 若客訴內容模糊無法判斷產品型號或問題細節，在對應欄位標注「⚠️ 需業務確認」
- 兩份檔案同時產出，不需使用者分開要求
